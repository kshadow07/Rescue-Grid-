# Volunteer Filtering & Map Rendering Enhancement Plan

## Current State Analysis

### 1. Database Schema & Indexing

**Current Tables:**
- `volunteer`: stores lat/lng as `double precision`, skills/equipment as `text` (comma-separated)
- `assignment`: connects volunteers to tasks
- `task_force_member`: junction table for team assignments

**Current Indexes** (`002_performance_indexes.sql`):
```sql
-- Status filtering (most common query)
CREATE INDEX idx_volunteer_status_active ON public.volunteer (status) WHERE status = 'active';

-- Name text search
CREATE INDEX idx_volunteer_name_lower ON public.volunteer (lower(name));

-- Composite index for geospatial queries
CREATE INDEX idx_volunteer_location ON public.volunteer (latitude, longitude) 
  WHERE latitude IS NOT NULL AND longitude IS NOT NULL;
```

**Problems Identified:**
1. ❌ No PostGIS - uses manual Haversine formula (SQL function `calculate_distance`)
2. ❌ Skills/equipment in comma-separated text - can't use indexes (uses `ILIKE` queries)
3. ❌ Distance sorting happens in application code after fetching records
4. ❌ No partial index for location-based queries combined with status

### 2. Volunteer Search API (`/api/volunteer/search/route.ts`)

**Current Flow:**
1. Bounding box filter (latDelta/lngDelta) using `idx_volunteer_location`
2. ILIKE filters for skills/equipment (full table scan on results)
3. Fetch records with limit (default 20, max 50)
4. Post-query: Calculate Haversine distance in JS for each result
5. Sort by distance in JS
6. Filter by radius in JS

**Performance Issues:**
- Bounding box can return many records before filtering by radius
- Skills/equipment ILIKE can't use indexes efficiently
- Distance calculation happens after fetch (can't limit results early)

### 3. Assignment Creation Scoring (`CreateAssignmentModal.tsx:212-237`)

**Current Scoring (Client-side only):**
```typescript
score = Math.max(0, 30 - distance_km * 2)           // Distance
        + matches.length * 15                        // Skills
        + equipmentMatches.length * 10               // Equipment
        + (status === "active" ? 10 : 0)            // Status bonus
        + (status === "on-mission" ? -20 : 0)       // Status penalty
```

**Problems:**
- Scoring happens AFTER fetching all volunteers
- Not using the API scoring - client-side only for task force creation
- No database-level ranking

### 4. Map Rendering (`MapboxMap.tsx`)

**Current Implementation:**
- Renders ALL volunteers from `useVolunteers()` hook
- No clustering - individual markers for every volunteer
- No zoom-based layer switching
- Real-time updates via Supabase subscriptions
- `volunteersVersion` state forces re-render on any change

**Problems:**
- Performance degrades linearly with volunteer count
- No heatmap/cluster view for overview
- No top-N limiting for dense areas

---

## Answers to Design Questions

1. **Scale:** < 1000 volunteers (lightweight solution sufficient)
2. **Skill Categories:** Use proposed hierarchy (MEDICAL, RESCUE, LOGISTICS, COMMUNICATION, SPECIAL)
3. **Tier System:** Global T1-T4 per volunteer (simplifies schema)
4. **Scoring Weights:** 0.40×skill + 0.35×proximity + 0.25×availability (fixed)
5. **Implementation:** Use composite index + DB Haversine (no PostGIS needed)

```sql
  ON volunteer_skills (skill_id, volunteer_id);

-- 5. DB-level scoring function
CREATE OR REPLACE FUNCTION calculate_volunteer_score(
  v_lat          FLOAT,
  v_lng          FLOAT,
  v_tier         SMALLINT,
  v_status       TEXT,
  v_last_seen    TIMESTAMPTZ,
  center_lat     FLOAT,
  center_lng     FLOAT,
  max_radius_km  FLOAT,
  required_skill_ids INT[]
) RETURNS FLOAT AS $$
DECLARE
  dist_km        FLOAT;
  proximity_score FLOAT;
  avail_score    FLOAT;
  skill_score    FLOAT;
BEGIN
  -- Haversine distance (km)
  dist_km := 6371 * 2 * ASIN(
    SQRT(
      POWER(SIN(RADIANS(v_lat - center_lat) / 2), 2) +
      COS(RADIANS(center_lat)) * COS(RADIANS(v_lat)) *
      POWER(SIN(RADIANS(v_lng - center_lng) / 2), 2)
    )
  );

  IF dist_km > max_radius_km THEN RETURN 0; END IF;

  -- Proximity: linear decay, closer = higher
  proximity_score := 1.0 - (dist_km / max_radius_km);

  -- Availability: active + seen recently scores highest
  avail_score := CASE
    WHEN v_status != 'active'                              THEN 0.0
    WHEN v_last_seen > NOW() - INTERVAL '15 minutes'      THEN 1.0
    WHEN v_last_seen > NOW() - INTERVAL '1 hour'          THEN 0.7
    WHEN v_last_seen > NOW() - INTERVAL '6 hours'         THEN 0.4
    ELSE 0.1
  END;

  -- Skill match: tier-weighted (0 if no skills required)
  skill_score := CASE
    WHEN required_skill_ids IS NULL OR array_length(required_skill_ids, 1) IS NULL
      THEN (v_tier::FLOAT / 4.0)
    ELSE (v_tier::FLOAT / 4.0)
  END;

  RETURN ROUND(
    (0.40 * skill_score + 0.35 * proximity_score + 0.25 * avail_score)::NUMERIC, 4
  );
END;
$$ LANGUAGE plpgsql IMMUTABLE;
```

```typescript
import { NextRequest, NextResponse } from 'next/server'
import { createServiceClient } from '@/lib/supabase/service'

interface SearchParams {
  latitude: number
  longitude: number
  radius_km?: number        // default 100
  skill_codes?: string[]    // e.g. ['first_aid', 'paramedic']
  limit?: number            // default 50, max 100
}

export async function POST(req: NextRequest) {
  if (process.env.USE_DB_SCORING !== 'true') {
    return NextResponse.json({ error: 'Feature not enabled' }, { status: 404 })
  }

  const supabase = createServiceClient()
  const body: SearchParams = await req.json()
  const {
    latitude,
    longitude,
    radius_km = 100,
    skill_codes = [],
    limit = 50,
  } = body

  // Resolve skill codes → IDs
  let skillIds: number[] = []
  if (skill_codes.length > 0) {
    const { data: rows } = await supabase
      .from('skill_definitions')
      .select('id')
      .in('code', skill_codes)
    skillIds = rows?.map((r: any) => r.id) || []
  }

  // Bounding box pre-filter
  const latDelta = radius_km / 111.0
  const lngDelta = radius_km / (111.0 * Math.cos((latitude * Math.PI) / 180))

  // Use raw query for the scoring function
  const query = `
    SELECT
      v.id,
      v.name,
      v.phone_number,
      v.latitude,
      v.longitude,
      v.tier,
      v.status,
      v.last_seen,
      ARRAY_AGG(sd.code) FILTER (WHERE sd.code IS NOT NULL) AS skills,
      calculate_volunteer_score(
        v.latitude, v.longitude, v.tier, v.status, v.last_seen,
        $1, $2, $3,
        $4
      ) AS score
    FROM volunteer v
    LEFT JOIN volunteer_skills vs ON vs.volunteer_id = v.id
    LEFT JOIN skill_definitions sd ON sd.id = vs.skill_id
    WHERE
      v.status = 'active'
      AND v.latitude  BETWEEN $1 - $5 AND $1 + $5
      AND v.longitude BETWEEN $2 - $6 AND $2 + $6
      ${skillIds.length > 0
        ? `AND EXISTS (
            SELECT 1 FROM volunteer_skills vs2
            WHERE vs2.volunteer_id = v.id
            AND vs2.skill_id = ANY($4)
          )`
        : ''
      }
    GROUP BY v.id, v.name, v.phone_number, v.latitude, v.longitude,
             v.tier, v.status, v.last_seen
    HAVING calculate_volunteer_score(
      v.latitude, v.longitude, v.tier, v.status, v.last_seen,
      $1, $2, $3, $4
    ) > 0
    ORDER BY score DESC
    LIMIT $7
  `

  const { data: volunteers, error } = await supabase.rpc('exec_sql', {
    sql: query,
    params: [latitude, longitude, radius_km, skillIds.length > 0 ? skillIds : null, latDelta, lngDelta, Math.min(limit, 100)]
  })

  if (error) {
    return NextResponse.json({ error: error.message }, { status: 500 })
  }

  return NextResponse.json({
    volunteers: volunteers || [],
    meta: {
      total: volunteers?.length || 0,
      center: { latitude, longitude },
      radius_km,
    },
  })
}
```

```typescript
import { NextRequest, NextResponse } from 'next/server'
import { createServiceClient } from '@/lib/supabase/service'

export async function GET(req: NextRequest) {
  const supabase = createServiceClient()
  const { searchParams } = new URL(req.url)
  const zoom = parseInt(searchParams.get('zoom') ?? '10')
  const lat = parseFloat(searchParams.get('lat') ?? '0')
  const lng = parseFloat(searchParams.get('lng') ?? '0')
  const bbox = searchParams.get('bbox')  // "minLng,minLat,maxLng,maxLat"

  let minLat = lat - 1, maxLat = lat + 1
  let minLng = lng - 1, maxLng = lng + 1

  if (bbox) {
    const [minLo, minLa, maxLo, maxLa] = bbox.split(',').map(parseFloat)
    minLng = minLo; minLat = minLa; maxLng = maxLo; maxLat = maxLa
  }

  if (zoom >= 12) {
    // Individual pins: top-50 scored only
    const { data: volunteers } = await supabase.rpc('get_top_scored_volunteers', {
      center_lat: lat,
      center_lng: lng,
      min_lat: minLat,
      max_lat: maxLat,
      min_lng: minLng,
      max_lng: maxLng,
      max_results: 50
    })

    return NextResponse.json({ type: 'pins', data: volunteers })
  }

  // Zoom < 12: return all active volunteers in bbox for client clustering
  const { data: volunteers } = await supabase
    .from('volunteer')
    .select(`
      id, latitude, longitude, tier,
      volunteer_skills(skill_definitions(category:skill_categories(code)))
    `)
    .eq('status', 'active')
    .gte('latitude', minLat)
    .lte('latitude', maxLat)
    .gte('longitude', minLng)
    .lte('longitude', maxLng)
    .limit(500)

  // Transform to get primary_category
  const transformed = volunteers?.map((v: any) => ({
    id: v.id,
    latitude: v.latitude,
    longitude: v.longitude,
    tier: v.tier,
    primary_category: v.volunteer_skills?.[0]?.skill_definitions?.category?.code || 'UNKNOWN'
  }))

  return NextResponse.json({ type: 'cluster_input', data: transformed })
}
```

```typescript
'use client'

import { useRef, useState, useEffect, useCallback } from 'react'
import { Marker } from 'react-map-gl/mapbox'
import Supercluster from 'supercluster'
import { debounce } from '@/lib/utils'

interface VolunteerMapLayerProps {
  map: mapboxgl.Map | null
  onVolunteerSelect?: (volunteer: any) => void
}

const superclusterRef = useRef<Supercluster | null>(null)

export function VolunteerMapLayer({ map, onVolunteerSelect }: VolunteerMapLayerProps) {
  const [zoom, setZoom] = useState(10)
  const [mapData, setMapData] = useState<{ type: string; data: any[] } | null>(null)
  const [clusters, setClusters] = useState<any[]>([])

  // Fetch from /api/volunteer/map
  const fetchMapData = useCallback(async () => {
    if (!map) return
    const bounds = map.getBounds()
    const z = Math.round(map.getZoom())
    const center = map.getCenter()

    const bbox = [
      bounds.getWest(), bounds.getSouth(),
      bounds.getEast(), bounds.getNorth()
    ].join(',')

    const res = await fetch(
      `/api/volunteer/map?zoom=${z}&lat=${center.lat}&lng=${center.lng}&bbox=${bbox}`
    )
    const json = await res.json()
    setMapData(json)
    setZoom(z)
  }, [map])

  // Rebuild Supercluster when cluster_input arrives
  useEffect(() => {
    if (!mapData || mapData.type !== 'cluster_input' || !map) return

    const sc = new Supercluster({
      radius: 60,
      maxZoom: 12,
      map: (props) => ({ tier: props.tier, category: props.category }),
      reduce: (acc, props) => {
        acc.count = (acc.count || 0) + 1
        acc.totalTier = (acc.totalTier || 0) + props.tier
        // Track dominant category
        if (!acc.categories) acc.categories = {}
        acc.categories[props.category] = (acc.categories[props.category] || 0) + 1
      }
    })

    sc.load(mapData.data.map((v: any) => ({
      type: 'Feature',
      geometry: { type: 'Point', coordinates: [v.longitude, v.latitude] },
      properties: { id: v.id, tier: v.tier, category: v.primary_category },
    })))

    superclusterRef.current = sc

    // Get initial clusters
    const bounds = map.getBounds()
    const initialClusters = sc.getClusters(
      [bounds.getWest(), bounds.getSouth(), bounds.getEast(), bounds.getNorth()],
      zoom
    )
    setClusters(initialClusters)
  }, [mapData, map, zoom])

  // Update clusters on zoom/move
  useEffect(() => {
    if (!map || !superclusterRef.current) return

    const updateClusters = () => {
      const bounds = map.getBounds()
      const z = Math.round(map.getZoom())
      const newClusters = superclusterRef.current!.getClusters(
        [bounds.getWest(), bounds.getSouth(), bounds.getEast(), bounds.getNorth()],
        z
      )
      setClusters(newClusters)
      setZoom(z)
    }

    const debouncedUpdate = debounce(updateClusters, 100)
    map.on('moveend', debouncedUpdate)
    map.on('zoomend', debouncedUpdate)

    return () => {
      map.off('moveend', debouncedUpdate)
      map.off('zoomend', debouncedUpdate)
    }
  }, [map])

  // Initial fetch
  useEffect(() => {
    fetchMapData()
  }, [fetchMapData])

  // Render based on zoom and data type
  if (!mapData) return null

  // Zoom > 12: individual scored pins
  if (mapData.type === 'pins') {
    return (
      <>
        {mapData.data.map((v: any) => (
          <Marker key={v.id} longitude={v.longitude} latitude={v.latitude}>
            <VolunteerPin
              tier={v.tier}
              score={v.score}
              skills={v.skills}
              onClick={() => onVolunteerSelect?.(v)}
            />
          </Marker>
        ))}
      </>
    )
  }

  // Zoom 10–12: Supercluster bubbles
  if (mapData.type === 'cluster_input' && clusters.length > 0) {
    return (
      <>
        {clusters.map((cluster) => {
          const [lng, lat] = cluster.geometry.coordinates
          if (cluster.properties.cluster) {
            const categories = cluster.properties.categories || {}
            const dominantCategory = Object.entries(categories)
              .sort((a: any, b: any) => b[1] - a[1])[0]?.[0] || 'UNKNOWN'

            return (
              <Marker key={`cluster-${cluster.id}`} longitude={lng} latitude={lat}>
                <ClusterBubble
                  count={cluster.properties.point_count}
                  dominantCategory={dominantCategory}
                  onClick={() => map?.flyTo({ center: [lng, lat], zoom: zoom + 2 })}
                />
              </Marker>
            )
          }
          return (
            <Marker key={cluster.properties.id} longitude={lng} latitude={lat}>
              <SmallDot tier={cluster.properties.tier} />
            </Marker>
          )
        })}
      </>
    )
  }

  // Zoom < 10: heatmap (handled by parent via Mapbox layer)
  return null
}
```

```typescript
export function debounce<T extends (...args: any[]) => void>(
  fn: T,
  delay: number
): (...args: Parameters<T>) => void {
  let timeoutId: ReturnType<typeof setTimeout>
  return (...args: Parameters<T>) => {
    clearTimeout(timeoutId)
    timeoutId = setTimeout(() => fn(...args), delay)
  }
}
```

```sql
-- Helper for map endpoint
CREATE OR REPLACE FUNCTION get_top_scored_volunteers(
  center_lat FLOAT,
  center_lng FLOAT,
  min_lat FLOAT,
  max_lat FLOAT,
  min_lng FLOAT,
  max_lng FLOAT,
  max_results INT DEFAULT 50
) RETURNS TABLE (
  id UUID,
  name TEXT,
  latitude FLOAT,
  longitude FLOAT,
  tier SMALLINT,
  status TEXT,
  skills TEXT[],
  score FLOAT
) AS $$
BEGIN
  RETURN QUERY
  SELECT
    v.id,
    v.name,
    v.latitude,
    v.longitude,
    v.tier,
    v.status,
    ARRAY_AGG(sd.code) FILTER (WHERE sd.code IS NOT NULL) AS skills,
    calculate_volunteer_score(
      v.latitude, v.longitude, v.tier, v.status, v.last_seen,
      center_lat, center_lng, 100, NULL
    ) AS score
  FROM volunteer v
  LEFT JOIN volunteer_skills vs ON vs.volunteer_id = v.id
  LEFT JOIN skill_definitions sd ON sd.id = vs.skill_id
  WHERE
    v.status = 'active'
    AND v.latitude  BETWEEN min_lat AND max_lat
    AND v.longitude BETWEEN min_lng AND max_lng
  GROUP BY v.id, v.name, v.latitude, v.longitude, v.tier, v.status, v.last_seen
  HAVING calculate_volunteer_score(
    v.latitude, v.longitude, v.tier, v.status, v.last_seen,
    center_lat, center_lng, 100, NULL
  ) > 0
  ORDER BY score DESC
  LIMIT max_results;
END;
$$ LANGUAGE plpgsql STABLE;

-- Helper for skill gaps endpoint
CREATE OR REPLACE FUNCTION get_skill_coverage_by_ring(
  center_lat FLOAT,
  center_lng FLOAT,
  ring_min_km FLOAT,
  ring_max_km FLOAT
) RETURNS TABLE (
  category TEXT,
  volunteer_count BIGINT,
  max_tier SMALLINT,
  avg_tier FLOAT
) AS $$
BEGIN
  RETURN QUERY
  SELECT
    sc.code AS category,
    COUNT(DISTINCT v.id) AS volunteer_count,
    MAX(v.tier) AS max_tier,
    AVG(v.tier)::FLOAT AS avg_tier
  FROM volunteer v
  JOIN volunteer_skills vs ON vs.volunteer_id = v.id
  JOIN skill_definitions sd ON sd.id = vs.skill_id
  JOIN skill_categories sc ON sc.id = sd.category_id
  WHERE v.status = 'active'
    AND (6371 * 2 * ASIN(SQRT(
      POWER(SIN(RADIANS(v.latitude - center_lat) / 2), 2) +
      COS(RADIANS(center_lat)) * COS(RADIANS(v.latitude)) *
      POWER(SIN(RADIANS(v.longitude - center_lng) / 2), 2)
    ))) BETWEEN ring_min_km AND ring_max_km
  GROUP BY sc.code
  ORDER BY sc.code;
END;
$$ LANGUAGE plpgsql STABLE;
```

```typescript
import { createServiceClient } from '@/lib/supabase/service'

const OLD_TO_NEW_SKILL_MAP: Record<string, string> = {
  'medical': 'first_aid',
  'rescue': 'rope_rescue',
  'first-aid': 'first_aid',
  'swimming': 'swimmer',
  'climbing': 'rope_rescue',
  'driving': 'driver',
  'communications': 'ham_radio',
  'logistics': 'driver',
  'search': 'structural_search',
  'firefighting': 'firefighter',
  'water-rescue': 'swimmer',
  'extraction': 'structural_search',
  'navigation': 'driver',
}

async function migrateSkills() {
  const supabase = createServiceClient()

  // Get all volunteers with skills
  const { data: volunteers } = await supabase
    .from('volunteer')
    .select('id, skills')
    .not('skills', 'is', null)

  if (!volunteers) {
    console.log('No volunteers with skills found')
    return
  }

  console.log(`Migrating ${volunteers.length} volunteers...`)

  for (const vol of volunteers) {
    const oldSkills = vol.skills.split(',').map((s: string) => s.trim().toLowerCase())
    const newSkillCodes = oldSkills
      .map((s: string) => OLD_TO_NEW_SKILL_MAP[s] || s)
      .filter(Boolean)

    if (newSkillCodes.length === 0) continue

    // Get skill IDs
    const { data: skillDefs } = await supabase
      .from('skill_definitions')
      .select('id, code')
      .in('code', newSkillCodes)

    if (!skillDefs || skillDefs.length === 0) continue

    // Insert volunteer_skills
    const inserts = skillDefs.map((sd: any) => ({
      volunteer_id: vol.id,
      skill_id: sd.id
    }))

    const { error } = await supabase
      .from('volunteer_skills')
      .upsert(inserts, { onConflict: 'volunteer_id,skill_id' })

    if (error) {
      console.error(`Failed to migrate volunteer ${vol.id}:`, error)
    } else {
      console.log(`Migrated ${vol.id}: ${newSkillCodes.join(', ')}`)
    }
  }

  console.log('Migration complete!')
}

migrateSkills().catch(console.error)
```
