## What Was Fixed

### Critical Bug: Volunteer COMPLETE Button
**Problem:** API expected `status` field, UI sent `action` field → 400 error on every status update.

**Fix:** API now accepts both `status` AND `action` (backward compatible).

### Critical Bug: Wrong Column Name
**Problem:** Code tried to update `victim_report_status` column which doesn't exist.

**Fix:** Now correctly updates `status` column and cascades:
- `on_my_way`/`arrived` → victim_report status = `in-progress`
- `completed` → victim_report status = `resolved`

### Critical Bug: Async Filter in Queue Route
**Problem:** `filter(async...)` returns Promise objects (truthy), so ALL items pass the filter.

**Fix:** Replaced with `for...of` loop with proper `await`.

### New: useMessages Hook
Built a reusable hook handling all 5 channel types with:
- Correct Supabase filter per channel type
- Realtime subscriptions with deduplication
- Optimistic message sending
- Mark-as-read functionality

### New: ChatScreen Component
Mobile-optimized chat UI with:
- `100dvh` flex layout (fixes keyboard overlap issue)
- `visualViewport` handling for iOS
- Auto-scroll to bottom
- Flag button toggle

### New: MessageBubble Component
Reusable bubble with:
- Own vs other message styling
- DMA message visual distinction
- Private reply `→ NAME` prefix
- Flagged message red border

### Fix: Hardcoded Channel Names
Fixed `"dma-messages-all"` and `"dma-topbar-counters"` to use unique names per session.

### Fix: DMA Counters
Now includes `on_my_way` and `arrived` assignments in the active count.
