# Implementation Plan: Offline SOS SMS Reporting via Twilio Webhook

## Executive Summary

Implement an offline-capable victim reporting feature where:
1. User taps SOS button → opens native SMS app with pre-filled Twilio number + structured message containing GPS coordinates
2. User sends SMS manually (works without internet)
3. Twilio receives SMS → forwards to Supabase Edge Function webhook
4. Edge Function parses SMS → creates victim_report in database

This bypasses the need for internet connectivity on the victim's device.

---

## 1. DATABASE SCHEMA ANALYSIS (victim_report table)

From `001_initial_schema.sql`, the victim_report table has these fields:
- **id** (uuid) - auto-generated
- **phone_no** (text, NOT NULL) - victim's phone
- **latitude** (double precision) - GPS
- **longitude** (double precision) - GPS
- **city** (text) - derived from reverse geocoding (optional)
- **district** (text) - derived from reverse geocoding (optional)
- **situation** (text, NOT NULL) - type: food/water/medical/rescue/shelter/missing
- **custom_message** (text) - optional additional details
- **urgency** (text) - default 'moderate'
- **status** (text) - default 'open'
- **created_at** (timestamp) - auto-generated
- **updated_at** (timestamp) - via trigger

### What the SMS must contain:
1. **phone_no** - from user's manual input (they have control)
2. **latitude** - from device GPS
3. **longitude** - from device GPS
4. **situation** - selected emergency type
5. **custom_message** - optional details

### What will be computed server-side:
- **city** and **district** from Mapbox reverse geocoding (in edge function)
- **urgency** can be auto-derived based on situation type

## 2. PROPOSED SMS FORMAT

### Human-Friendly + Machine-Parseable Format:

```
RESCUEGRID SOS
Phone: +91XXXXXXXXXX
Location: 28.6129,77.2295
Type: rescue
Msg: 6 people stuck on roof, water rising
```

### Why this format:
1. **Human-readable**: Users can edit/modify before sending
2. **Structured**: Edge function can reliably parse using regex/key-value pairs
3. **Flexible**: If user deletes some lines, we can still parse with fallbacks
4. **Fits SMS limits**: 160 chars per segment, this format is compact

### SMS Structure Details:
```
Line 1: "RESCUEGRID SOS" (identifier for our system)
Line 2: "Phone: +[country][number]" 
Line 3: "Location: [latitude],[longitude]"
Line 4: "Type: [food|water|medical|rescue|shelter|missing]"
Line 5+: "Msg: [custom message]" (optional, can span multiple lines)
```

## 3. IMPLEMENTATION COMPONENTS

### Component A: SOS SMS Button with Geolocation

**File**: `rescuegrid/app/(victim)/page.tsx` (Victim Home Page)

**New Component**: Create a reusable SMS button component at `components/victim/SOSSMSButton.tsx`

**Flow**:
1. User selects emergency type (food/water/medical/rescue/shelter/missing)
2. Taps new "🚨 SOS (No Internet)" button
3. Prompts for phone number (if not cached in localStorage)
4. Requests device GPS via `navigator.geolocation.getCurrentPosition()`
5. Constructs SMS body with formatted message
6. Opens native SMS app using `sms:` URI scheme

**SMS URI Scheme Implementation**:
```typescript
// Cross-platform SMS URI (handles iOS, Android variations)
const phoneNumber = process.env.NEXT_PUBLIC_TWILIO_SMS_NUMBER; // e.g., +16045551234
const message = encodeURIComponent(`RESCUEGRID SOS\nPhone: ${userPhone}\nLocation: ${lat},${lng}\nType: ${type}\nMsg: ${customMsg || 'None'}`);

// Try modern format first (?body=), fall back to older format (;body=)
const smsUrl = `sms:${phoneNumber}?body=${message}`;
// Fallback for older Android: `sms:${phoneNumber};body=${message}`

window.location.href = smsUrl;
```

**Key Technical Details**:
- `navigator.geolocation` API works even in airplane mode if GPS chip is available
- iOS: `sms:+123?body=text` (works in Safari)
- Android: `sms:+123?body=text` (modern) or `sms:+123;body=text` (older)
- Use `window.location.href` assignment for best compatibility
- Show loading state while GPS is being acquired

### Component B: Supabase Edge Function (twilio-sms-webhook)

**File**: `supabase/functions/twilio-sms-webhook/index.ts`

**Trigger**: HTTP POST from Twilio webhook (configured in Twilio Console)

**Function Flow**:
1. Receive form-encoded webhook from Twilio
2. Verify the request is from Twilio (optional but recommended)
3. Parse SMS body fields:
   - `From` → sender's phone number
   - `Body` → the message content
4. Extract structured data from Body:
   - Parse "Phone:" line → victim phone
   - Parse "Location:" line → lat,lng
   - Parse "Type:" line → situation
   - Parse "Msg:" line → custom_message
5. Call Mapbox reverse geocoding API → get city, district
6. Insert into `victim_report` table
7. Return TwiML `<Response/>` (empty, Twilio needs any valid response)

**Twilio Webhook Payload Format**:
```typescript
{
  From: "+91XXXXXXXXXX",       // Sender's phone
  To: "+16045551234",          // Your Twilio number
  Body: "RESCUEGRID SOS\nPhone: ...",  // The SMS text
  SmsSid: "SM...",             // Unique message ID
  SmsStatus: "received",
  FromCountry: "IN",
  // Other fields...
}
```

**Edge Function Response** (required by Twilio):
```xml
<Response/>
```

## 5. KEY TECHNICAL RESEARCH FINDINGS

### SMS URI Scheme Research
- **iOS Safari**: `sms:+123?body=message` ✓
- **Android Chrome**: `sms:+123?body=message` ✓ (Android 4+)
- **Older Android**: `sms:+123;body=message` (pre-4.0, rare now)
- **Limitations**:
  - Does NOT work on desktop browsers (no SMS app)
  - On mobile, opens whatever default SMS app user has
  - User can edit message before sending ✓ (this is required feature)

### Twilio Incoming SMS Webhook
- **Content-Type**: `application/x-www-form-urlencoded`
- **Method**: POST
- **Key Fields**:
  - `From`: Sender's E.164 formatted phone number (+1234567890)
  - `To`: Your Twilio number
  - `Body`: SMS text content
  - `SmsSid`: Unique identifier
  - `FromCountry`: ISO country code

### Service Worker Offline Detection
- **Events**: `online`, `offline` fire on window/navigator
- **navigator.onLine**: Returns boolean, but doesn't guarantee internet (could be local network)
- **Better check**: Try fetching a small resource or use the online/offline events as approximation
- **PWA Install**: Need manifest.json + service worker for proper PWA behavior

## 7. EDGE CASES & ERROR HANDLING

### Client-Side (SOS Button):
- **GPS unavailable** → Show error: "Please enable location services"
- **Phone number empty** → Prompt user to enter
- **SMS not supported** → Show alert: "Please send SMS manually to [number]"
- **User edits message before sending** → Edge function handles partial/malformed messages with regex fallbacks

### Server-Side (Edge Function):
- **Message format malformed** → Try to extract phone from Twilio's `From` field, use regex to find coordinates
- **Invalid coordinates** → Return 200 (so Twilio doesn't retry), log error
- **Duplicate SMS** → Check for existing report from this phone within last 5 minutes

### Twilio Configuration:
- **Webhook not responding** → Twilio retries with exponential backoff
- **Invalid TwiML response** → Message shows "not delivered" to user
- **Signature verification failed** → Log and ignore (optional security)

## 8. SECURITY CONSIDERATIONS

1. **SMS Spoofing**: Anyone can send SMS to your Twilio number
   - Mitigation: Verify phone format, rate limit per number
   - Could add simple auth code in future enhancement

2. **Edge Function Exposure**: Publicly accessible URL
   - Could add Twilio signature verification (recommended)
   - Check `X-Twilio-Signature` header
   - Use Twilio SDK's validateRequest helper

3. **Phone Number Privacy**: Store in database, don't expose in logs

4. **Coordinates Display**: DMA dashboard uses Mapbox to show markers from lat/lng coordinates

## 9. ALTERNATIVE APPROACHES CONSIDERED

### Alternative 1: Full PWA Offline Queue
- Use IndexedDB to queue reports
- Sync when connection restored
- Rejected: Doesn't solve immediate emergency need when internet is down for extended periods

### Alternative 2: SMS-to-Email Gateway
- Use free SMS-to-email services
- Rejected: Unreliable, not real-time

### Alternative 3: Third-party SMS API
- Use services like TextMagic, Nexmo
- Rejected: Adds complexity, Twilio already integrated for OTP

### Selected Approach (Native SMS + Twilio Webhook):
- ✅ Works with zero internet on victim device
- ✅ Uses existing Twilio number
- ✅ Fast and reliable
- ✅ Can be done from any phone (even feature phones if they support SMS)

## APPENDIX A: SUPABASE EDGE FUNCTION CODE (Ready to Copy)

### File: `supabase/functions/twilio-sms-webhook/index.ts`

```typescript
// supabase/functions/twilio-sms-webhook/index.ts
import { serve } from "https://deno.land/std@0.168.0/http/server.ts";
import { createClient } from "https://esm.sh/@supabase/supabase-js@2.38.4";

const SUPABASE_URL = Deno.env.get("SUPABASE_URL") || "";
const SUPABASE_SERVICE_ROLE_KEY = Deno.env.get("SUPABASE_SERVICE_ROLE_KEY") || "";

// CORS headers for Twilio
const corsHeaders = {
  "Access-Control-Allow-Origin": "*",
  "Access-Control-Allow-Methods": "POST, OPTIONS",
  "Content-Type": "text/xml",
};

interface ParsedSMS {
  phone: string;
  latitude: number | null;
  longitude: number | null;
  situation: string;
  customMessage: string | null;
}

function parseSMSBody(body: string, fromPhone: string): ParsedSMS {
  const result: ParsedSMS = {
    phone: fromPhone, // fallback to Twilio's From number
    latitude: null,
    longitude: null,
    situation: "rescue", // default
    customMessage: null,
  };

  // Try to extract phone from "Phone:" line
  const phoneMatch = body.match(/Phone:\s*([\+\d\s\-\(\)]+)/i);
  if (phoneMatch) {
    result.phone = phoneMatch[1].replace(/\s/g, "");
  }

  // Try to extract location from "Location:" line
  const locationMatch = body.match(/Location:\s*([\d\.\-]+)\s*,\s*([\d\.\-]+)/i);
  if (locationMatch) {
    result.latitude = parseFloat(locationMatch[1]);
    result.longitude = parseFloat(locationMatch[2]);
  }

  // Try to extract type from "Type:" line
  const typeMatch = body.match(/Type:\s*(\w+)/i);
  if (typeMatch) {
    const type = typeMatch[1].toLowerCase();
    const validTypes = ["food", "water", "medical", "rescue", "shelter", "missing"];
    if (validTypes.includes(type)) {
      result.situation = type;
    }
  }

  // Try to extract message from "Msg:" line
  const msgMatch = body.match(/Msg:\s*(.+)(?:\n|$)/is);
  if (msgMatch) {
    result.customMessage = msgMatch[1].trim();
    if (result.customMessage === "None" || result.customMessage === "N/A") {
      result.customMessage = null;
    }
  }

  return result;
}

serve(async (req: Request) => {
  // Handle preflight
  if (req.method === "OPTIONS") {
    return new Response("ok", { headers: corsHeaders });
  }

  try {
    // Parse form data from Twilio
    const formData = await req.formData();
    const from = formData.get("From") as string;
    const to = formData.get("To") as string;
    const body = formData.get("Body") as string;
    const smsSid = formData.get("SmsSid") as string;

    console.log(`Incoming SMS from ${from} to ${to}: ${body}`);

    // Parse the SMS content
    const parsed = parseSMSBody(body, from);

    // Validate required fields
    if (!parsed.latitude || !parsed.longitude) {
      console.error("Missing coordinates in SMS");
      return new Response(
        `<?xml version="1.0" encoding="UTF-8"?><Response><Message>Could not process: Missing location data</Message></Response>`,
        { status: 200, headers: corsHeaders }
      );
    }

    // Create Supabase client with service role
    const supabase = createClient(SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY);

    // Check for duplicate report in last 5 minutes
    const fiveMinutesAgo = new Date(Date.now() - 5 * 60 * 1000).toISOString();
    const { data: existing } = await supabase
      .from("victim_report")
      .select("id")
      .eq("phone_no", parsed.phone)
      .gte("created_at", fiveMinutesAgo)
      .limit(1);

    if (existing && existing.length > 0) {
      console.log("Duplicate report detected, skipping");
      return new Response(
        `<?xml version="1.0" encoding="UTF-8"?><Response/>`,
        { status: 200, headers: corsHeaders }
      );
    }

    // Determine urgency based on situation
    const urgencyMap: Record<string, string> = {
      rescue: "critical",
      medical: "critical",
      missing: "urgent",
      food: "urgent",
      water: "urgent",
      shelter: "moderate",
    };

    // Insert victim report (city/district will be displayed on DMA dashboard using Mapbox)
    const { data: report, error } = await supabase
      .from("victim_report")
      .insert({
        phone_no: parsed.phone,
        latitude: parsed.latitude,
        longitude: parsed.longitude,
        situation: parsed.situation,
        custom_message: parsed.customMessage,
        status: "open",
        urgency: urgencyMap[parsed.situation] || "moderate",
      })
      .select("id")
      .single();

    if (error) {
      console.error("Database error:", error);
      return new Response(
        `<?xml version="1.0" encoding="UTF-8"?><Response><Message>Error saving report</Message></Response>`,
        { status: 200, headers: corsHeaders }
      );
    }

    console.log(`Created victim_report: ${report.id}`);

    // Return empty TwiML response (success)
    return new Response(
      `<?xml version="1.0" encoding="UTF-8"?><Response/>`,
      { status: 200, headers: corsHeaders }
    );

  } catch (err) {
    console.error("Edge function error:", err);
    return new Response(
      `<?xml version="1.0" encoding="UTF-8"?><Response/>`,
      { status: 200, headers: corsHeaders }
    );
  }
});
```
